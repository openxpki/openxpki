Current actions/ToDo

The roadmap is only a very rough description of the next steps. Here you can find more detailed informations what's actually going on.

  • Integration of a software engine with new configuration options - engine_usage and key_store (Julia)
  • Rewrite the OpenSSL shell usage and move it towards a normal CLI usage - use OpenSSL's engine section (Michael)
  • System bootstrap workflows (database init etc.) (Martin Bartosch)
  • Basic PKI workflows (request, issuance, CRL issuance) (Martin Bartosch)
  • Shibboleth support
  • Command line client (Martin Bartosch)
  • Web Interface: Mason (Martin Bartosch)
  • SCEP Interface (Martin Bartosch)

Wishlist/Sometimes maybe

These topics might be addressed in the future.

  • Native XS implementation of Certificate Issuance (instead of using the OpenSSL command line interface).
  • Workflow configuration history (pending workflow instances remember the configuration that was active when the WF instance was created).
  • Cron-like time based event trigger (either integrated in the server or triggered by the CLI client)
  • Packaging for various Unix platforms
